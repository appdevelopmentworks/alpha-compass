# alpha-compass

日本市場ファースト × 米国強化の **投資情報統合モニター**（デスクトップアプリ）。

World Monitor の金融レイヤーを蒸留し、「網羅しすぎて分かりにくい」という弱点を情報設計で逆手に取り、日本語化かつ日本市場に合わせて再構築する。**World Monitor のコードは一切流用せず**、挙動・機能設計の発想のみを参照するクリーンルーム実装。

## 状態
- フェーズ: scaffold 準備完了（実装は Claude Code セッションで進行）
- ライセンス/権利: 自前実装。参照元 World Monitor は AGPL-3.0（© Elie Habib）だがコードは非流用。

## スタック
- シェル: **Tauri 2**（Rust）
- フロント: **Next.js（静的エクスポート） + TypeScript**、チャートは lightweight-charts / Recharts(visx)
- データ/AI サイドカー: **Python（uv）+ FastAPI**
- 永続層: **DuckDB**（時系列/分析）＋ **SQLite**（設定/状態）
- ローカル LLM: Qwen MoE（RTX 5090）、上位は Claude API（コストカスケード）

## 設計の二大原則
1. **情報設計ファースト** — 網羅より厳選 / 1パネル1問い / 結論→根拠。
2. **クリーンルーム** — World Monitor のコード・アセット・定義を含まない。

## アーキテクチャ要点
- **Rust(Tauri Core) が DuckDB/SQLite の唯一の所有者（単一ライター）**。計算・スケジューリング・アラート・IPC を担う。
- **Python サイドカー** は外部データ取得＋AI推論のステートレス・サービス（正規化 JSON を Rust に返すのみ）。
- **フロントは外部 API を直接叩かない**（Rust 経由のみ。CORS/鍵漏洩を回避）。
- 詳細は `docs/アーキテクチャ設計書.md`。

## ドキュメント
- `docs/要件定義書.md` — 要件・スコープ・設計思想
- `docs/アーキテクチャ設計書.md` — 構成・データモデル・IPC・地合いスコア仕様・AIカスケード
- `docs/セッションプラン.md` — v1 のセッション分割（Session 0〜5）
- `docs/CLAUDE_CODE_キックオフ.md` — 各セッションの Claude Code プロンプト（英語）

## 始め方
1. `docs/CLAUDE_CODE_キックオフ.md` の **Session 0** プロンプトを Claude Code に投入し、骨組み（Tauri + Next + サイドカー疎通）を作る。
2. 以降 Session 1〜5 を順に実行（米国コア → 地合いコンポジット＋ホーム → 日本市場 → 開示AI＋アラート → クロスマーケット＋仕上げ）。
3. 各セッションは `docs/セッションプラン.md` の受け入れ基準で確認。

## データソース（v1）
- 日本: **J-Quants（無料プラン／配信遅延あり・鮮度表示）**、EDINET、TDnet
- 米国/グローバル: yfinance、FRED、CFTC COT、USD/JPY
- ※ 四季報オンラインのスクレイピングは ToS 違反のため不可

## 注意
- 本アプリの地合いスコアやクロスマーケット注記は **状況把握のための情報提示**であり、売買推奨ではない。
- API トークンはキーチェーン管理。`.env` は開発用途のみで Git 管理外。

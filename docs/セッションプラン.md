# alpha-compass セッションプラン

> プロジェクト: **alpha-compass**
> ステータス: 確定 v1.0（session-plan フェーズ）
> 前提ドキュメント: `docs/要件定義書.md` / `docs/アーキテクチャ設計書.md`
> 目的: v1 を Claude Code で実装するための **セッション単位の作業分割**。各セッションは独立した成果物（受け入れ基準）を持つ。

---

## 進め方

- 各セッションは Claude Code に `docs/CLAUDE_CODE_キックオフ.md` の対応プロンプトを投入して開始する。
- セッション冒頭で必ず `docs/要件定義書.md` と `docs/アーキテクチャ設計書.md` を読ませる。
- **クリーンルーム厳守**: World Monitor のコード/アセット/定義は参照・流用しない（挙動と機能設計の発想のみ）。
- **言語ポリシー**: アプリ UI は日本語。コード内コメント・型定義・エージェント/プロンプト・コミットメッセージは英語。
- 各セッション終了時に受け入れ基準を満たしたかを確認し、`docs/` に簡単な進捗メモを追記する。

---

## モデル運用の指針（参考）

## Session 0 — プロジェクト初期化（骨組み）

**目的**: Tauri 2 + Next.js(静的エクスポート) + Python サイドカー(FastAPI) の動く骨組みを作り、フロント↔Rust↔サイドカーの疎通を確認する。

**スコープ / 主な作業**
- 既存の `docs/` を温存したまま、`アーキテクチャ設計書 §13` のディレクトリ構造を作る。
- 公式ツールで Tauri 2 + Next.js(TS) を初期化（`output: 'export'` を設定）。`src/` と `src-tauri/` を生成。
- `sidecar/` に `uv` で FastAPI プロジェクトを初期化。`/health` エンドポイントを実装。
- Tauri から Python サイドカーを spawn する配線（external binary もしくは開発時は `uv run`）。起動時にセッショントークンを環境変数で渡す（§4.3）。
- Rust に `duckdb` / `rusqlite` を導入し、空の DB を初期化できることを確認。
- 疎通確認用の最小 IPC: フロントの「ping」ボタン → Rust → サイドカー `/health` → 結果表示。

**触るファイル（主）**
- `package.json`, `next.config.js`, `tsconfig.json`
- `src-tauri/Cargo.toml`, `src-tauri/tauri.conf.json`, `src-tauri/src/main.rs`, `src-tauri/src/sidecar/`
- `sidecar/pyproject.toml`, `sidecar/app/main.py`
- `.gitignore`, `.env.example`（ルートに既存。必要なら追記）

**受け入れ基準**
- `dev` 起動でアプリが立ち上がる。
- 「ping」でフロント→Rust→サイドカー→フロントの往復が成功する。
- DuckDB/SQLite ファイルが生成される。

## Session 1 — データ基盤 ＋ 米国コア

**目的**: 永続層とスケジューラの骨格を作り、**米国市場コア**を実データで表示する。

**スコープ / 主な作業**
- Store 層: DuckDB スキーマ（`prices`, `indices`, `rates_macro`, `us_breadth`, `sector_perf`, `cot` ほか）と SQLite（`settings`, `source_meta`）のマイグレーション。
- Scheduler 骨格（tokio）: ソース別 cadence の枠、`source_meta` 鮮度更新、バックオフ。
- サイドカー Adapters: `yfinance`（米株・指数）, `fred`（金利・クレジット・DXY）, `cot`。Normalizer で共通スキーマへ。
- Rust が取得 JSON を upsert。
- IPC: `get_us_market()`, `get_freshness()`, `refresh(source?)`。
- フロント「米国市場」ビュー（最小）: 指数、セクター相対強弱、ブレッドス、金利/VIX/DXY、COT。鮮度表示付き。

**受け入れ基準**
- 米国コアが実データで表示され、手動 `refresh` で更新される。
- 各パネルに最終取得時刻が出る。

## Session 2 — 地合いコンポジット ＋ ホーム

**目的**: 説明可能な地合いスコアを計算し、ホーム画面で「結論」を一望できるようにする。

**スコープ / 主な作業**
- Compute Engine（Rust, 純関数中心）: 指標（200DMA 乖離・傾き、ブレッドス集計）、正規化（rolling z-score / winsorize）、加重和、レジーム判定（§8）。
- `composite_scores` / `signal_states` への書込み。寄与度を JSON で保持。
- IPC: `get_composite()`。
- フロント「ホーム」ビュー: 地合いスコア(大) ＋ 寄与度バー ＋ 主要指数ミニ ＋（ブリーフ枠はプレースホルダ）＋ アラート枠（後続）。
- ユニットテスト: コンポジット計算。

**受け入れ基準**
- ホームに 0–100 のスコアとレジーム、各構成要素の寄与度が表示される。
- 重みは設定値から読み込む（既定は §8）。

## Session 3 — 日本市場モジュール

**目的**: J-Quants(無料) を中心に **日本市場**を表示し、海外勢フローを地合いコンポジットへ統合する。

**スコープ / 主な作業**
- サイドカー Adapter: `jquants`（無料プラン）。株価四本値・指数・**投資部門別**・信用週末残・空売り比率。**配信遅延を前提**にし、鮮度ラベルへ反映。
- 為替: USD/JPY 取得。先物（OSE/CME/SGX）から寄付ギャップ示唆。
- Store: `jp_investor_flows`, `jp_margin`, `jp_short_selling`, `fx_rates`。
- IPC: `get_jp_market()`, `get_watchlist()`/`set_watchlist()`。
- フロント「日本市場」ビュー: 指数・業種ヒートマップ、投資部門別フロー、信用/空売り、先物・ギャップ、ドル円、ウォッチリスト。
- コンポジットへ「海外勢フロー」シグナルを接続（週次・前値保持・鮮度明示）。

**受け入れ基準**
- 日本市場ビューが表示され、海外勢フローがコンポジット寄与度に現れる。
- J-Quants 無料の遅延がユーザに分かる形で明示される。

## Session 4 — 開示AI ＋ アラート

**目的**: 開示/ニュースの日本語 AI 要約（コストカスケード）と、ノイズの少ないアラートを実装する。

**スコープ / 主な作業**
- サイドカー Adapters: `edinet`（有報・大量保有等）, `tdnet`（適時開示、取得方式を確定）, `news`（日本語金融ニュース RSS/API）, `calendar`（経済・決算・FOMC・日銀）。
- AI Router（§10）: Tier1 ルール（XBRL 定型抽出）→ Tier2 ローカル Qwen（自由文要約）→ Tier3 Claude API（当日ブリーフ）。結果を DB にキャッシュ、`summarized_tier` 記録、Tier3 は日次予算ゲート。
- Store: `disclosures`, `news`, `calendar_events`, `alerts`, `alert_rules`。
- Alert Engine（§11）: 複数ソース照合、重複排除、レート制限。`alert:fired` イベント。
- IPC: `get_disclosures(filter)`, `get_alerts()`, `get_alert_rules()`。
- フロント「開示/ニュース」ビュー（フィルタ付き）、「アラート」ビュー（履歴＋ルール）、ホームにブリーフ要約と最新アラートを反映。

**受け入れ基準**
- 開示/ニュースが日本語要約付きで表示され、生成 tier が分かる。
- 複数シグナル一致時のみアラートが発火し、履歴に残る。

## Session 5 — クロスマーケット ＋ 仕上げ

**目的**: 米国→日本トランスミッションを実装し、設定・鮮度・オフライン・エラー処理を仕上げて v1 完成。

**スコープ / 主な作業**
- Compute: クロスマーケット・ルール（§9, 編集可）。`cross_market` 書込み。IPC `get_cross_market()`。フロント「クロスマーケット」ビュー。
- 「設定」ビュー: 資格情報（キーチェーン経由 `set_credential`）、更新間隔、ウォッチリスト、コンポジット重み調整、レイアウト。
- 横断: 鮮度 UI の磨き込み、オフライン時のキャッシュ表示、エラーハンドリング、ログ（取得失敗・レート制限・tier 使用）。
- （任意）⌘K コマンドパレット。

**受け入れ基準**
- クロスマーケットの示唆が表示される。
- 設定から各種調整ができ、資格情報がキーチェーンに保存される。
- オフライン/取得失敗時もキャッシュで破綻しない。
- v1 スコープ（要件定義 §3.1）を満たす。

## セッション横断のチェックリスト

- [ ] World Monitor のコードを参照していない（クリーンルーム）。
- [ ] フロントは外部 API を直接叩いていない（Rust 経由のみ）。
- [ ] 資格情報は平文保存していない（キーチェーン）。
- [ ] DuckDB の書込みは Rust のみ（単一ライター）。
- [ ] 各データに鮮度（最終取得時刻）が付与・表示される。
- [ ] 地合いスコアは寄与度を開示し説明可能。
- [ ] UI は日本語、コード/プロンプトは英語。

## 1. システム全体構成

4プロセス（論理）構成。配布は単一の Tauri アプリにまとまる。

```
┌──────────────────────────────────────────────────────────────────┐
│                      alpha-compass (Tauri 2 App)                    │
│                                                                    │
│  ┌────────────────────────┐         ┌──────────────────────────┐  │
│  │   Frontend (WebView)    │  invoke │     Tauri Core (Rust)     │  │
│  │  Next.js static export  │◄───────►│  - IPC コマンドハンドラ   │  │
│  │  TypeScript             │  events │  - Compute Engine(指標)   │  │
│  │  lightweight-charts     │         │  - Scheduler (tokio)      │  │
│  │  Recharts / visx        │         │  - Alert Engine           │  │
│  │  7ビュー(後述)          │         │  - Store I/O              │  │
│  └────────────────────────┘         └───────────┬──────────────┘  │
│                                                  │ owns             │
│                                   ┌──────────────┴──────────────┐  │
│                                   │  DuckDB(時系列/分析)         │  │
│                                   │  SQLite(設定/状態)           │  │
│                                   │  OS Keychain(資格情報)        │  │
│                                   └──────────────┬──────────────┘  │
│                                                  │ HTTP(127.0.0.1)  │
│                                   ┌──────────────┴──────────────┐  │
│                                   │   Python Sidecar (FastAPI)   │  │
│                                   │  - Adapters(各データ源)       │  │
│                                   │  - Normalizer                │  │
│                                   │  - AI Router(コストカスケード)│  │
│                                   └──────────────┬──────────────┘  │
└──────────────────────────────────────────────────┼────────────────┘
                                                    │ 外部 API
        J-Quants / EDINET / TDnet / yfinance / FRED / CFTC COT /
        為替 / 経済・決算カレンダー / 金融ニュース / Claude API
```

**役割分担の原則**
- **Tauri Core (Rust)** が司令塔。DuckDB/SQLite の **唯一の所有者（単一ライター）**。計算・スケジューリング・アラート・IPC を担う。
- **Python Sidecar** は「外部データ取得 ＋ AI 推論」のステートレス・サービス。永続化はせず、正規化済み JSON を Rust に返すだけ（キャッシュの正本は DuckDB）。
- **Frontend** は表示専用。Rust への `invoke` と event 購読のみでデータ取得する（直接外部 API を叩かない＝CORS/鍵漏洩を回避）。

この topology により、(1) DuckDB の同時書込み競合を構造的に排除、(2) 資格情報を Rust 側に集約、(3) Python のデータ/AI ライブラリ資産を最大活用、を同時に満たす。

## 2. データフロー

```
[Scheduler(Rust)] ──tick──► [Sidecar 呼出(HTTP)] ──► [Adapter が外部API取得]
        │                                                      │
        │                                              [Normalizer で正規化]
        ▼                                                      │
[Rust が JSON 受領] ◄──────────────────────────────────────────┘
        │
        ├─► [DuckDB に raw/正規化テーブルを upsert]
        │
        ├─► [Compute Engine: 指標・地合いコンポジット・クロスマーケット・レジーム]
        │           │
        │           └─► [derived テーブルへ書込み]
        │
        ├─► [Alert Engine: signal_states を評価 → 照合 → alerts]
        │
        └─► [Tauri event 発火] ──► [Frontend が該当ビューを更新]
```

オンデマンド（ユーザ操作）時も同経路。Frontend の `invoke('get_*')` は **derived/正規化テーブルからの読取り**を返し、必要なら裏で sidecar リフレッシュを起動する（stale-while-revalidate）。

## 3. プロセス詳細

### 3.1 Tauri Core (Rust)
- **IPC ハンドラ** (`ipc/`): フロントエンド公開コマンド（第4章）。
- **Compute Engine** (`engine/`): 指標計算（移動平均・トレンド・ブレッドス集計等）、地合いコンポジット（第8章）、クロスマーケット（第9章）、レジーム判定。純関数中心でユニットテスト可能に。
- **Scheduler** (`scheduler/`): tokio タスク。ソース別 cadence（第7章）でサイドカー取得を駆動。レート制限・指数バックオフ・鮮度更新。
- **Alert Engine** (`alerts/`): signal_states を評価し複数ソース照合（第11章）。
- **Store** (`store/`): DuckDB（`duckdb` crate）＋ SQLite（`rusqlite`）。マイグレーション管理。
- **Sidecar Manager** (`sidecar/`): 起動時に Python サイドカーを spawn、ヘルスチェック、HTTP クライアント、セッショントークン管理（第4.3章）。

### 3.2 Python Sidecar (FastAPI / uv 管理)
- **Adapters** (`adapters/`): ソースごとに 1 モジュール（`jquants.py`, `edinet.py`, `tdnet.py`, `yfinance.py`, `fred.py`, `cot.py`, `fx.py`, `news.py`, `calendar.py`）。取得・ページング・リトライをカプセル化。
- **Normalizer** (`normalize/`): 各ソースの生レスポンスを共通スキーマ（第5章）に変換。単位・タイムゾーン（→JST）・銘柄コード正規化。
- **AI Router** (`ai/`): コストカスケード（第10章）。`router.py` が tier 判定、`rules.py`（Tier1）、`local_qwen.py`（Tier2, RTX 5090）、`claude.py`（Tier3）。
- 配布: 開発は `uv`、リリースは PyInstaller で単一バイナリ化し **Tauri の sidecar（external binary）** として同梱。

### 3.3 Frontend (Next.js static export + TypeScript)
- `next.config.js` は `output: 'export'`（SSR なし）。Tauri webview が静的成果物を読み込む。
- Tauri 連携: `lib/ipc.ts` に `invoke` ラッパと event 購読を集約。
- チャート: 価格系は **lightweight-charts**、コンポジット/寄与度/フローは **Recharts もしくは visx**。
- 表示整形: `lib/format.ts` に日本式数値（円・万・億・%）、JST 時刻、市場時間判定。
- 状態管理: 軽量に（Zustand 等）。サーバ状態は invoke 結果＋event で同期。

## 4. IPC 設計

### 4.1 Frontend ↔ Rust（Tauri commands / events）
コマンド（同期取得、`serde` で型付き JSON 返却）:

| コマンド | 返却 | 用途 |
| --- | --- | --- |
| `get_composite()` | スコア・レジーム・寄与度配列 | ホームの地合い |
| `get_jp_market()` | 指数・業種・部門別・信用・空売り・先物・ドル円 | 日本市場ビュー |
| `get_us_market()` | 指数・セクター・ブレッドス・金利・VIX・DXY・COT・FedWatch | 米国ビュー |
| `get_cross_market()` | 伝播経路リスト | クロスマーケット |
| `get_disclosures(filter)` | 開示/ニュース要約フィード | 開示ビュー |
| `get_alerts()` / `get_alert_rules()` | アラート履歴・ルール | アラートビュー |
| `get_watchlist()` / `set_watchlist(..)` | ウォッチリスト | 設定/日本市場 |
| `get_freshness()` | ソース別最終取得時刻 | 鮮度表示 |
| `refresh(source?)` | ack | 手動更新 |
| `get_settings()` / `set_settings(..)` | 設定 | 設定 |
| `set_credential(source, token)` | ack（Keychain へ） | 資格情報登録 |

イベント（Rust → Frontend、push）:
- `data:updated`（payload: 更新ドメイン）/ `alert:fired`（payload: alert）/ `composite:updated` / `sidecar:status`。

### 4.2 Rust ↔ Python Sidecar（localhost HTTP）
- ベース: `http://127.0.0.1:<port>`（port は spawn 時に動的確保し Rust が保持）。
- エンドポイント例: `GET /health`、`POST /fetch/{source}`（body にパラメータ）、`POST /ai/summarize`、`POST /ai/brief`。
- 返却は正規化済み JSON（第5章スキーマ準拠）。

### 4.3 サイドカーのセキュリティ
- Rust が起動時に **ランダムなセッショントークン**を生成し、環境変数で sidecar に渡す。以後の全リクエストに `Authorization` ヘッダを付与し、sidecar は不一致を 401。→ 同一ホスト上の他プロセスからの不正アクセスを防止。
- bind は `127.0.0.1` のみ（外部公開しない）。

## 8. 地合いコンポジット算出仕様

> 本スコアは **状況把握のための透明な合成指標**であり、売買推奨ではない。重み・構成要素は設定で変更可能で、寄与度を必ず開示して「なぜこの値か」を説明可能にする。

**算出手順**
1. 構成シグナル `xᵢ` を取得（下表）。
2. 各 `xᵢ` を rolling 基準（過去 N 営業日の z-score もしくはパーセンタイル、±3σ で winsorize）で **正規化** → `nᵢ ∈ [−1, +1]`。リスクオンが正になるよう **符号** を付与。
3. 加重和 `A = Σ wᵢ·nᵢ`（`Σ wᵢ = 1`）。
4. `score = clamp(round(50 + 50·A), 0, 100)`。
5. レジーム判定（5段階）: `[0,20) 強リスクオフ / [20,40) リスクオフ / [40,60) 中立 / [60,80) リスクオン / [80,100] 強リスクオン`。
6. 各 `寄与度ᵢ = wᵢ·nᵢ` を `composite_scores.components` に保存・表示。

**構成シグナル（デフォルト）**

| シグナル | ソース/算出 | 符号 | デフォルト重み |
| --- | --- | --- | --- |
| 米国株トレンド | S&P500 の 200DMA 乖離・傾き | + | 0.20 |
| 市場ブレッドス | %above 200DMA・騰落 | + | 0.15 |
| ボラティリティ | VIX（rolling 中央値比、反転） | − | 0.15 |
| クレジット | HY OAS（基準比、反転） | − | 0.15 |
| 金利 | 米10年の変化（**符号は設定可**＝局面依存） | 可変 | 0.10 |
| ドル円 | USD/JPY トレンド（円安=日本株リスクオン寄り、設定可） | + | 0.10 |
| 海外勢フロー | 投資部門別 海外 net（週次） | + | 0.15 |

- 金利の符号は「急騰＝ストレス」局面と「景気期待＝リスクオン」局面で意味が反転するため、既定は中立寄せ＋ユーザ設定で切替可能とする。
- 週次シグナル（海外勢フロー）は次の公表まで前値を保持し、寄与度にその旨を明示。J-Quants 無料プランの遅延も鮮度表示で明示する。

## 9. クロスマーケット・トランスミッション

> 予測ではなく **透明なルールベースの注記**。米国時間外の動きと為替から、東証寄り付きで意識されやすいセクター傾斜を情報提示する。ルールは編集可能。

入力: 前日 US 終値・夜間先物・米金利・ドル円。出力: `(driver, path, effect_note)` のリスト。

ルール例（既定・編集可）:
- `米10年↑ & USD/JPY↑` → 金融・輸出株に追い風 / グロース・不動産に逆風
- `VIX↑ & ナスダック↓` → 日本のハイグロース・半導体に逆風
- `WTI↑` → 資源・商社に追い風 / 空運・電力に逆風

表示は「観測された前提 → 想定される傾斜」を簡潔に1行で。あくまで示唆であることを UI 上で明示。

## 10. AI インテリジェンス層（コストカスケード）

| Tier | 実装 | 用途 | コスト |
| --- | --- | --- | --- |
| Tier1 ルール | テンプレート抽出 | XBRL 決算の定型抽出（売上/利益/前年比）、定型開示の見出し整形 | 0 |
| Tier2 ローカル | Qwen MoE on RTX 5090（ONNX Runtime / ローカル推論） | ニュース・開示自由文の日本語要約、ルーチンな当日メモ | GPU のみ |
| Tier3 Claude API | Anthropic API | 多数シグナル横断の当日ブリーフ、含意の深い要約 | 従量（ゲート） |

- **ルーティング**: 重要度・文書長・構造化可否で tier 判定。Tier1 で完結するものは LLM を呼ばない。
- **キャッシュ**: 要約結果は `disclosures.summary` / `news.summary` / ブリーフ表に保存し再計算を回避。`summarized_tier` で生成元を記録。
- Tier3 は日次予算上限・対象を絞り、暴発を防ぐ。

## 11. アラート・エンジン

- **ルール** = `signal_states` のしきい値判定の論理結合 ＋ **照合要件**（独立な N 系統以上が一致して初めて発火）。
  - 例: 「海外勢 売り越し」 AND 「USD/JPY 急落」 AND 「VIX 急騰」 が同時 → severity=high。
- **重複排除/レート制限**: 同一 alert key を時間窓内で抑制。
- 発火 → `alerts` テーブル ＋ `alert:fired` イベント → フロントのトースト/履歴。
- 通知の外部連携（Slack/Telegram 等）は v2（要件定義 第3.2）。

## 12. フロントエンド構成（7ビュー）

情報設計原則に従い、各ビューは目的を絞る。

1. **ホーム（Overview）**: 地合いスコア（大）＋寄与度バー＋主要指数ミニ＋当日ブリーフ要約＋最新アラート。1画面で結論。
2. **日本市場**: 指数・業種ヒートマップ、投資部門別フロー、信用/空売り、先物・寄付ギャップ、ドル円。
3. **米国市場**: 指数・セクター相対強弱、ブレッドス、金利/クレジット/VIX/DXY、COT、利上げ確率。
4. **クロスマーケット**: 米国→日本トランスミッション。
5. **開示/ニュース**: AI 要約フィード、フィルタ（銘柄/種別/重要度）。
6. **アラート**: 履歴とルール設定。
7. **設定**: ソース資格情報（キーチェーン）、更新間隔、ウォッチリスト、コンポジット重み、レイアウト。

- コマンドパレット（⌘K）は v1.5/任意（実装容易だがコア機能ではない）。
- 全パネルに **鮮度（最終取得時刻）** を表示。stale はトーンダウン。

## 14. 非機能・横断事項

- **鮮度**: 全データに `ingested_at`/`source_meta`。UI に最終取得時刻、閾値超過で視覚的にデグレード表示。
- **オフライン**: 外部取得失敗時は DuckDB キャッシュを表示し「最終取得時刻」を明示。
- **セキュリティ**: 資格情報はキーチェーン。サイドカーはローカル限定＋トークン認証。フロントは外部 API を直接叩かない。
- **パフォーマンス**: 起動〜地合い表示まで数秒。計算は Rust で高速、重い取得はサイドカー非同期。
- **テスト**: Compute Engine は純関数中心でユニットテスト。アダプタは記録レスポンスでテスト。
- **可観測性**: ログ（取得失敗・レート制限・tier 使用状況）。

## 15. 技術選定の根拠（要約）

| 選定 | 理由 |
| --- | --- |
| Tauri 2 デスクトップ | 既存スタック。ローカル単体動作・配布容易・GPU 活用と相性良 |
| Rust が DB 所有 + 計算 | 単一ライターで競合排除、高速な指標計算、型安全 |
| Python サイドカー | 金融データ/AI ライブラリ資産（yfinance・各種パーサ・LLM）を活用 |
| DuckDB + SQLite | 時系列/分析は DuckDB、設定/状態は SQLite（用途分離） |
| コストカスケード AI | ルール→ローカル Qwen→Claude で品質とコストを両立、RTX 5090 を活用 |
| lightweight-charts | 金融チャートの定番、軽量・高速 |

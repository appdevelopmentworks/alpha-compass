## 1. 背景と目的

### 1.1 背景
World Monitor は完成度の高いリアルタイム・グローバルインテリジェンス基盤であり、金融バリアント（finance.worldmonitor.app）も持つ。ただし設計思想が「地政学・軍事・衛星・海底ケーブル・市場…を1画面に網羅」する situational awareness 全部入りであり、**投資判断に使う観点では情報過多で可読性が低い**。

### 1.2 目的
World Monitor の **金融に関係する部分のみを抽出・蒸留** し、以下を満たす投資情報モニターを自前スタックで構築する。

- 投資に関する情報（株式・為替・金利・コモディティ・マクロ・ニュース/開示）を **1つの画面で俯瞰** できる
- 米国市場をベースとしつつ、**米国市場をさらに強化**（セクターローテーション、ブレッドス、クレジット、ポジショニング等）
- **日本語化 + 日本市場対応**（東証・日経・TOPIX・日本固有シグナル）
- World Monitor より **明確で分かりやすい**（網羅しすぎない、情報階層を持つ）

### 1.3 非目的（このプロジェクトで作らないもの）
- 売買シグナルの自動執行 / 発注（モニタリング専用。エントリー判定は alpha-radar 側の責務）
- 価格予測モデルの学習・推論（neuro-candle / prediction-eval の責務）
- 地政学・軍事・OSINT 全般の網羅（金融に効くマクロ/供給シグナルに限定）

## 2. 設計思想（最重要）

World Monitor の "網羅しすぎ" に対するアンチテーゼとして、**情報設計を機能要件と同格の第一級要件**として扱う。

1. **1パネル＝1つの問い。** 各パネルは「今日の地合いは？」「海外勢は買っているか？」「米金利はどちらに振れているか？」のように単一の問いに答える。
2. **トップに "結論"、下に "根拠"（プログレッシブ・ディスクロージャ）。** 起動直後に当日の総合判断（リスクオン/オフ・地合いスコア）を一望、ドリルダウンで詳細へ。
3. **網羅より厳選。** 数百フィードを並べない。意思決定に効くシグナルを絞る。
4. **地図は主役ではなく補助。** World Monitor は地図ファーストだが、本プロジェクトはシグナル/データファースト。地図は「資本フロー」「セクターヒートマップ」等の補助ビューに限定。
5. **日本語ファーストの情報設計。** 数値は円・万・億・％の日本式表記、時刻は JST、市場時間（東証 9:00–11:30 / 12:30–15:30、米国はサマータイム対応）を前提にしたUI。
6. **ノイズの少ないアラート。** 単一ソースでは鳴らさず、複数の独立シグナルが一致したときのみ通知（World Monitor の corroboration 思想を踏襲）。

## 4. 機能要件

### 4.1 マーケット・コンポジット（地合いスコア）
複数シグナルを合成した単一スコアで「今日の地合い」を表現する（World Monitor の 7-signal composite に相当する蒸留版）。

- 構成シグナル候補: 米国株トレンド、ブレッドス、VIX、クレジットスプレッド、ドル円方向、米金利方向、海外投資家フロー（日本）
- 出力: 0–100 のリスクオン/オフ指標 ＋ 各構成要素の寄与度（説明可能であること）
- 計算は **Rust エンジン**（alpha-radar と同様の指標エンジン設計を参考に新規実装）

### 4.2 日本市場モジュール
- 指数: 日経225 / TOPIX / 東証グロース市場指数、業種別騰落（TOPIX-17, 33業種）
- **投資部門別売買動向**: 海外投資家・個人・信託・事業法人等の週次売買差。海外勢フローは日経の主要ドライバーであり、本プロジェクトの差別化シグナルとして最重要視する。
- 信用残: 買い残/売り残/信用倍率の推移、空売り比率
- 先物: 日経225先物（OSE / CME / SGX）。夜間の海外取引から **翌日寄り付きのギャップ示唆** を表示
- 個別銘柄: ウォッチリスト（株価四本値・出来高・主要財務）

### 4.3 米国市場モジュール（強化領域）
World Monitor 比で深掘りする。
- 指数 + GICS/SPDR 11セクターの相対強弱（**セクターローテーション**、RRG 的表示は v2）
- 市場ブレッドス: 騰落レシオ、新高値–新安値、200日線上比率
- 金利/クレジット: 2s10s カーブ、HY OAS、DXY、VIX
- 流動性: FRB バランスシート・RRP（マクロ流動性、v2 で拡充可）
- ポジショニング: CFTC COT（新規実装）
- 金融政策: FOMC スケジュール、利上げ/利下げ確率

### 4.4 クロスマーケット（米国→日本トランスミッション）
World Monitor の「相関レイヤー（市場の動きには地図がある）」を、投資文脈に絞って再構成する中核機能。
- 米国市場の前日終値・夜間先物・ドル円から、**東証寄り付きへの波及** を可視化
- 例: 「米10年金利↑ → ドル円↑ → 輸出セクター追い風」のような伝播経路を簡潔に提示

### 4.5 ニュース・適時開示インテリジェンス
- 日本語金融ニュース ＋ 適時開示（TDnet）/ EDINET 開示を収集
- **AI 要約**（日本語）: コストカスケード方式（ルールベース → ローカル Qwen MoE on RTX 5090 → Claude API）。alpha-sentinel と同種の思想を本プロジェクト内に新規実装。
- 当日ブリーフ: 引用元付きで「今日の注目点」を生成

### 4.6 アラート
- 複数の独立シグナルが一致したときのみ発火（例: 海外勢売り越し ＋ ドル円急落 ＋ VIX 急騰）
- レート制限・重複排除でノイズ抑制

### 4.7 補助地図ビュー（v2・任意）
- 資本フロー / グローバル・セクターヒートマップ（新規実装の地図ビュー）

### 4.8 エージェント連携（v2・任意）
- MCP サーバー化し、ACI / alpha-sentinel 等の自前エージェントから本モニターのデータを参照可能に

## 6. 非機能要件

- **ローカルファースト**: Tauri デスクトップアプリとして単体動作。クラウド常時接続を前提にしない。
- **GPU 活用**: ローカル LLM 推論（Qwen MoE）に RTX 5090 を使用。ONNX Runtime in Rust も選択肢。
- **更新頻度**: 市場時間中は指数/為替を準リアルタイム（数十秒〜分）、開示/部門別は配信頻度に準拠（日次/週次）。
- **データ層**: 時系列・分析クエリは DuckDB、アプリ設定/状態は SQLite。
- **オフライン耐性**: 最終取得データのキャッシュ表示。ソースごとの鮮度（タイムスタンプ）を明示。
- **可読性/応答性**: 起動から地合いスコア表示まで数秒。パネル切替はストレスフリー。

## 7. アーキテクチャ方針（概要）

詳細は次フェーズだが、標準スタックへのマッピングを先に固定する。レイヤー構成:

1. **データ取込（Python サイドカー / uv）**: J-Quants・EDINET・TDnet（日本）＋ yfinance・FRED・COT（米国）＋ 為替・カレンダー・ニュース。FastAPI サイドカー（qwen-tts-desktop の構成を踏襲）。
2. **正規化・蓄積（DuckDB）**: 統一時系列 ＋ リファレンスデータ。
3. **計算エンジン（Rust）**: 地合いコンポジット、テクニカル、クロスマーケット・シグナル、レジーム判定。alpha-radar と同様の設計を参考に新規実装。
4. **インテリジェンス層（コストカスケード LLM）**: 日本語要約・開示サマリ・アラート照合。
5. **提示層（Next.js 静的エクスポート + Tauri 2）**: 厳選ダッシュボード。チャートは lightweight-charts（価格）+ Recharts/visx（コンポジット）想定。
6. **（任意）MCP/API サーフェス**: 自前エージェント連携（v2）。

## 9. 既存資産との関係（決定済み: 完全独立の新規実装）

**決定**: 本プロジェクトは **完全独立の新規実装** とする。alpha-sentinel / Global Flow Radar / alpha-radar 等の既存プロジェクトと **コードベースを共有せず**、データ/AI バックエンドも本プロジェクト専用に新規構築する。

ただし「完全独立」とは *コードの非共有* を意味し、Shin が確立した **設計パターンの踏襲は妨げない**。具体的には以下を独立実装の指針とする:

- データ取込 / AI 要約パイプライン: alpha-sentinel と同種の多層・コストカスケード思想を **本プロジェクト内に新規実装**（共有・依存はしない）
- 計算エンジン: alpha-radar と同様の Rust 指標エンジン設計を **参考にしつつ新規実装**
- 資本フロー / セクタービュー: Global Flow Radar の発想を参考に **新規実装**

これにより、外部依存を最小化した自己完結アプリとして配布・保守できる。重複実装のコストは受け入れる前提とする。
